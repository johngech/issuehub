# AGENTS.md — issue-tracker

## Project structure

Monorepo with three packages, all using **Bun** as runtime/package manager.

| package | dir | stack |
|---------|-----|-------|
| server | `server/` | Express 5, Bun runtime |
| client | `client/` | React 19, TanStack Router (file-based), Tailwind v4, Vite, Vitest |
| core | `core/` | Shared Zod schemas, types & constants (no runtime) |

Workspace config in root `package.json` (`"workspaces": ["client", "server", "core"]`) enables cross-package imports via `workspace:*`. Bun also auto-discovers via `bun.lock`.

## Commands

Run all commands from the relevant package directory.

### Root (utility scripts)
```
bun run dev:client          # cd client && bunx vite dev
bun run dev:server          # cd server && bun run --watch index.ts
bun run build:client        # cd client && bunx vite build
bun run build:server        # cd server && bun run index.ts
bun run format              # biome format --write (all packages)
bun run lint                # biome lint --write (all packages)
bun run check               # biome check --write (all packages)
```

### Biome config

Single `biome.json` at project root with `overrides` for per-package formatting:
- **Client**: space indent (width 2), double quotes
- **Server**: space indent, double quotes
- **Core**: tab indent, double quotes

## Key gotchas

- **Do NOT edit** `client/src/routeTree.gen.ts` — auto-generated by `@tanstack/router-plugin/vite`, regenerated on dev/build.
- Path aliases `#/` and `@/` both resolve to `client/src/`.
- `routeTree.gen.ts` and `styles.css` are excluded from Biome checking (configured in root `biome.json`).
- Server `src/` directory is currently empty; the app entrypoint is `server/index.ts`.
- `core/constants.ts` and `core/types.ts` are empty placeholders — populate as needed.
- `core/package.json` exports map is missing `./constants` — must be added before importing from `@issue-tracker/core/constants`.
- No CI/CD workflows present yet.
- **CORS**: Server reflects any origin (`origin: true`) with credentials — acceptable for local dev only. Harden before deployment.
- **Committed secrets**: `server/.env` was committed to git before `.gitignore` took effect. See review for remediation steps.
